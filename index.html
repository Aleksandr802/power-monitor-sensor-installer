🔄 Firmware Update (OTA)

Choose your preferred OTA method. The device must already have this firmware installed.

Recommended Method 1 — Update via USB (ESP Web Tools)

Same as first install — connects via USB cable and re-flashes only the firmware partition.

WiFi Method 2 — Over-the-Air via Local Network

Open the device's built-in OTA update page on your local network. Device must be connected to WiFi.

Find the IP address in your router's admin page or from the device serial output.

📦 What gets updated?

  1. 1Only the application partition (firmware.bin) is replaced — bootloader and settings are preserved.
  2. 2Your device's WiFi credentials and configuration are not erased.
  3. 3The device reboots into the new firmware automatically after update.
🔌 Serial port not found / no device listed
  • Try a different USB cable — many cables are charge-only and have no data lines.
  • Install the CH340 / CP210x driver for your OS (ESP32-C3 Super Mini uses CH340G).
  • On Linux, add yourself to the dialout group: sudo usermod -aG dialout $USER, then log out and back in.
  • Close all serial monitors (Arduino IDE, minicom, screen).
⚠️ Flashing starts but fails or times out
  • Hold the BOOT button on the ESP32-C3 Super Mini while clicking Install, release after the progress starts.
  • Try a shorter USB cable or a different port on your computer.
  • Reduce the baud rate if you see frequent errors (this is handled automatically by ESP Web Tools).
  • Ensure the device is not in deep sleep — press the reset button first.
🌐 Browser says "WebSerial not supported"
  • Use Google Chrome 89+ or Microsoft Edge 89+ on Windows, macOS, or Linux desktop.
  • Firefox, Safari, and mobile browsers do not support WebSerial.
  • Check that chrome://flags/#enable-experimental-web-platform-features is enabled if using an older Chrome version.
💡 Device connected but not responding after install
  • Press the RST (reset) button on the board to force a reboot.
  • Open a serial monitor at 115200 baud to see boot messages.
  • If the device bootloops, re-flash using the First Install tab (full flash).
